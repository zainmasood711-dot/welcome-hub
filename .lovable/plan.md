## الهدف
تحويل القالب الحالي من صفحة Placeholder إلى تطبيق ويب عملي يعمل فعلياً ببيانات محفوظة وقابل للتشغيل مباشرة.

## الافتراض التنفيذي (بسبب عدم تحديد مجال)
سنبدأ بـ **MVP لإدارة المهام** لأنه أسرع مسار لتحويل المشروع إلى منتج كامل يمكن استخدامه فوراً.

## نطاق النسخة الأولى
- واجهة تطبيق حقيقية بدل الصفحة الفارغة
- إدارة مهام كاملة (إضافة، تعديل، تغيير حالة، حذف)
- بحث + فلترة حسب الحالة والأولوية
- تخزين دائم للبيانات عبر Lovable Cloud
- Metadata أساسية محسّنة (title/description/og)

## خطة التنفيذ
### 1) تهيئة المنتج والواجهة
- استبدال `src/routes/index.tsx` بواجهة تشغيل فعلية (لوحة مهام).
- تنظيم الشاشة إلى: Header + أدوات سريعة + قائمة مهام + مؤشرات مختصرة.
- الالتزام بـ design tokens الحالية في `src/styles.css`.

### 2) تفعيل الخلفية (Lovable Cloud)
- تفعيل Lovable Cloud للمشروع.
- إنشاء جدول المهام مع صلاحيات وصول مناسبة.
- بناء دوال السيرفر (`createServerFn`) للقراءة والكتابة.

### 3) ربط البيانات بالواجهة
- جلب البيانات بنمط TanStack Query (`ensureQueryData` / `useSuspenseQuery` عند الحاجة).
- تنفيذ CRUD كامل عبر `useServerFn` + `useMutation` + invalidation صحيح.
- إضافة حالات Empty/Loading/Error وتجربة Retry.

### 4) تحسين تجربة الاستخدام
- فلاتر (todo / in_progress / done).
- بحث نصي مباشر.
- ترتيب (الأحدث / الاستحقاق / الأولوية).
- رسائل نجاح/فشل مختصرة.

### 5) الجاهزية للتشغيل
- تحديث metadata الافتراضية في الجذر والصفحة الرئيسية.
- اختبار التدفقات الأساسية من البداية للنهاية.
- مراجعة responsive على desktop/mobile.

## معايير القبول (Done)
- لا يوجد أي Placeholder في الصفحة الرئيسية.
- CRUD يعمل بالكامل مع حفظ دائم.
- البحث والفلترة يعملان بشكل صحيح.
- الواجهة مستقرة ومتجاوبة.
- التطبيق جاهز للمعاينة الفعلية والتشغيل.

## ما سيتم بعد الموافقة مباشرة
- تنفيذ الخطة خطوة بخطوة في نفس المشروع.
- تشغيل النسخة العملية في المعاينة والتأكد من عملها قبل التسليم.
