## ما سيتم تنفيذه الآن (فقط المرحلة الأولى)

سألتزم بتنفيذ **المرحلة الأولى فقط** دون أي بناء للمرحلة الثانية وما بعدها.

### 1) قاعدة البيانات والبنية الأمنية
- إنشاء جداول المرحلة الأولى فقط:
  - `profiles`
  - `engineers`
  - `product_categories`
  - `brands`
  - `products`
  - `error_codes`
- إضافة القيود المطلوبة (PK/FK/UNIQUE/CHECK/DEFAULT).
- تفعيل RLS على هذه الجداول فقط.
- إعداد سياسات الوصول حسب الأدوار:
  - `support_engineer`: CRUD كامل لهذه الوحدات.
  - `field_engineer`: صلاحيات قراءة محدودة حسب دوره في هذه المرحلة.
  - `manager`: قراءة فقط.
- إضافة فهارس محسنة للاستعلامات الأساسية في هذه المرحلة.
- إضافة triggers لـ `updated_at`.

### 2) المصادقة والأدوار والملفات الشخصية
- تفعيل Lovable Cloud للمصادقة وقاعدة البيانات.
- إنشاء تدفق تسجيل دخول عربي RTL باستخدام **بريد/هاتف + كلمة مرور**.
- ربط المستخدم بجدول `profiles` وتخزين الدور وربط `engineer_id` عند الحاجة.
- حراسة المسارات حسب الدور مع إعادة توجيه تلقائية بعد تسجيل الدخول.

### 3) واجهات المرحلة الأولى فقط
- **صفحة تسجيل الدخول** (عربية RTL، عملية، واضحة).
- **لوحة/تنقل حسب الدور**:
  - Support Engineer: وصول كامل لوحدات المرحلة الأولى.
  - Field Engineer: واجهة محدودة الصلاحية.
  - Manager: واجهة قراءة فقط.
- **إدارة المهندسين**:
  - قائمة + بحث + فلاتر + إضافة + تعديل + تفعيل/تعطيل.
- **إدارة الفئات/العلامات/المنتجات**:
  - CRUD كامل مع العلاقة: فئة ← علامة ← منتج.
- **إدارة رموز الأخطاء**:
  - CRUD + ربط اختياري بمنتج + تصنيف software/technical + بحث.

### 4) جودة وتجربة الاستخدام (ضمن المرحلة الأولى)
- تطبيق RTL كامل عبر التطبيق.
- توحيد المصطلحات العربية.
- تصميم Dashboard إداري نظيف ومهني للأجزاء المنفذة.
- Responsive (موبايل/ديسكتوب) مع حالات تحميل/خطأ/فراغ.
- تحقق مدخلات قوي Client + Server (Zod).

### 5) بيانات تجريبية (Seed) للمرحلة الأولى
- فئات: Battery / Inverter / Panel / Other.
- علامات: Pylontech / Goodwe / Growatt / JA Solar / Deye.
- منتجات: النماذج المذكورة.
- رموز أخطاء: 24 / 17 / 48.
- مستخدمون تجريبيون: دعم + ميداني + مدير (مع أدوار واضحة في profiles).

## ما لن يتم بناؤه الآن
- أي جزء من المرحلة الثانية أو الثالثة أو الرابعة أو الخامسة.
- لا Customers, لا Systems, لا Tickets, لا Assignments, لا KB, لا Attachments المتقدمة بعد.

## التسليم قبل الانتقال للمرحلة التالية
قبل أي انتقال للمرحلة التالية، سأعرض لك:
1. الصفحات المنفذة في المرحلة الأولى.
2. سير العمل الفعلي (Login → Role Redirect → CRUD للوحدات الأساسية).
3. ما تم بناؤه وما تبقى بالضبط.

## معيار قبول هذه المرحلة
- تسجيل الدخول يعمل بالأدوار.
- الصلاحيات مطبقة فعليًا على الواجهة والخادم وRLS.
- وحدات: Engineers + Categories/Brands/Products + Error Codes تعمل CRUD بالكامل.
- الواجهة عربية RTL وقابلة للاستخدام الفوري.
- لا وجود لأي تنفيذ لمراحل لاحقة.
